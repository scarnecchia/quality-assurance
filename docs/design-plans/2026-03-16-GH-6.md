# Streaming SAS-to-Parquet Conversion Design

## Summary

The cross-table validation engine currently converts SAS input files to Parquet by reading all chunks into memory and concatenating them before writing — meaning peak memory scales with the full size of the source file. This design replaces that approach with a streaming write: each chunk is cast to a canonical schema and written directly to disk via `pyarrow.parquet.ParquetWriter` as it arrives, keeping memory bounded to a single chunk at a time regardless of dataset size.

The canonical schema is derived from the SCDM specification already embedded in the codebase, mapping SCDM's two column types (`Numeric` → Arrow `float64`, `Character` → Arrow `utf8`) so that type consistency is enforced at write time rather than inferred post-hoc. Custom tables not in the spec fall back to first-chunk inference with a logged warning. The change is additive — a `table_key` parameter is threaded through to the conversion function; the temp file lifecycle, chunk reading pattern, and cleanup logic are all left intact.

## Definition of Done
- `_convert_sas_to_parquet()` streams chunks directly to a Parquet file via `pyarrow.parquet.ParquetWriter` instead of collecting them in memory
- Memory usage stays O(chunk_size) regardless of total file size
- All existing cross-table validation tests continue to pass with no behavioural change
- A new test verifies multi-chunk streaming write produces correct output

## Glossary

- **SCDM (Sentinel Common Data Model)**: A standardised schema for representing health data in the Sentinel distributed database network. Defines the tables, columns, types, and constraints this tool validates against.
- **Parquet**: A columnar binary file format optimised for analytical workloads. Used here as an intermediate format for cross-table validation.
- **Row group**: Parquet's internal unit of storage. Each streaming chunk written by `ParquetWriter` becomes one row group, enabling independent reads.
- **pyarrow / `ParquetWriter`**: Apache Arrow's Python implementation. `ParquetWriter` writes Parquet files incrementally via `write_table()` calls, replacing the full in-memory concat.
- **pyreadstat**: A Python library for reading SAS7BDAT files. The existing SAS reader wraps it to yield chunks.
- **SAS7BDAT**: A proprietary binary file format produced by SAS statistical software. The source format for SCDM data files being converted.
- **Canonical schema**: The authoritative `pyarrow.Schema` derived from the SCDM spec, as opposed to types inferred by pyreadstat from data. Prevents type drift across chunks.
- **`ColumnDef` / `TableSchema`**: Frozen dataclasses in this codebase representing SCDM column and table definitions respectively.
- **L2 / cross-table validation**: The second validation phase, checking relationships across multiple SCDM tables. SAS-to-Parquet conversion is a pre-processing step in this phase.
- **Fallback path**: When a `table_key` is not in the SCDM spec, schema is inferred from the first chunk rather than failing hard.

## Architecture

Replace the in-memory `pl.concat(chunks)` pattern in `_convert_sas_to_parquet()` with a streaming `pyarrow.parquet.ParquetWriter` that sinks each chunk to disk as it arrives.

**Data flow:**
1. Caller passes `table_key` (new parameter) alongside `sas_path` and `chunk_size`
2. Function looks up `TableSchema` via `get_schema(table_key)` and builds a canonical `pyarrow.Schema` from the `ColumnDef` definitions (SCDM `Numeric` → `float64`, `Character` → `utf8`)
3. Creates a `NamedTemporaryFile` for the output Parquet path
4. Opens `pyarrow.parquet.ParquetWriter(tmp_path, canonical_schema)` as a context manager
5. Iterates `reader.chunks()`, converting each `polars.DataFrame` to `pyarrow.Table` via `.to_arrow()`, casting to the canonical schema, and calling `writer.write_table(table)`
6. Each chunk becomes one Parquet row group
7. Writer closes on context manager exit (writes Parquet footer)
8. Logs conversion with column types and total row count

**Schema resolution:**
- Primary path: canonical schema built from SCDM spec (`get_schema(table_key)`)
- Fallback path: if `table_key` is not in the spec (custom tables), infer schema from first chunk and cast subsequent chunks to it. Log a warning that canonical schema was unavailable.

**Contract change:** `_convert_sas_to_parquet(sas_path, chunk_size, table_key)` — adds `table_key: str` parameter. Caller in `run_cross_table_checks` already has this value.

## Existing Patterns

The codebase already follows these patterns that this design builds on:

- **Chunked reading protocol:** `TableReader.chunks()` yields `polars.DataFrame` instances. The SAS reader (`src/scdm_qa/readers/sas.py`) uses `pyreadstat.read_file_in_chunks`. This design consumes chunks identically.
- **Schema lookup:** `get_schema(table_key)` is the established way to get SCDM table definitions. Already used in `_handle_length_excess` within the same file.
- **Lazy imports:** Format-specific libraries are imported inside functions (e.g., `create_reader` in the readers package). `pyarrow.parquet` will follow this pattern.
- **Temp file lifecycle:** `run_cross_table_checks` already tracks temp Parquet files in `temp_parquet_files` and cleans them up in a `finally` block. No change needed.

No divergence from existing patterns.

## Additional Considerations

**SCDM type mapping specifics:** The SCDM spec uses two types: `Numeric` (maps to `float64` in Arrow — SAS numeric is always floating-point) and `Character` (maps to `utf8`). The `length` field in `ColumnDef` is informational for validation purposes but does not affect the Arrow type (Arrow `utf8` is variable-length). Nullability is derived from `ColumnDef.missing_allowed`.

**Columns in data but not in spec:** SAS files may contain columns not defined in the SCDM spec (partner-specific additions). These should be preserved in the output Parquet using their pyreadstat-inferred types — the canonical schema applies only to spec-defined columns. The builder should produce a merged schema: canonical types for known columns, inferred types for unknown columns.
