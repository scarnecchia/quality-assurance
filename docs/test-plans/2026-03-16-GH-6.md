# Human Test Plan: Streaming SAS-to-Parquet Conversion (GH-6)

## Prerequisites

- Python 3.12+ with `uv` installed
- Project cloned and on the GH-6 branch
- `uv run pytest` passes (434 passed, 3 skipped)
- Access to a directory containing real SCDM `.sas7bdat` files for integration verification

## Phase 1: Memory Boundedness (AC2.1)

This is the sole criterion requiring human verification. The concern is that `_convert_sas_to_parquet` does not accumulate chunks in memory.

| Step | Action | Expected |
|------|--------|----------|
| 1 | Open `src/scdm_qa/validation/cross_table.py` and locate the `_convert_sas_to_parquet` function | Function exists |
| 2 | Search the function body for `pl.concat`, list accumulation patterns (e.g., `chunks.append`, `all_data = []`), or any variable that grows with each iteration | None found -- each chunk should be written to `ParquetWriter.write_table()` and then fall out of scope |
| 3 | Confirm the loop structure is: `for chunk in reader.chunks() -> cast -> writer.write_table(arrow_table)` with no intermediate collection | Pattern confirmed |
| 4 | Verify `writer.close()` is called after the loop (or the writer is used in a context manager) | Writer is properly closed |

## Phase 3: CLI End-to-End (Cross-Table with SAS Input)

Purpose: Validate that the full CLI pipeline handles SAS-to-Parquet conversion transparently when a user provides `.sas7bdat` files in config.

| Step | Action | Expected |
|------|--------|----------|
| 1 | Create a TOML config file with at least one table pointing to a `.sas7bdat` file, e.g.: `[tables]` / `demographic = "/path/to/demographic.sas7bdat"` / `enrollment = "/path/to/enrollment.parquet"` | Config created |
| 2 | Run: `uv run scdm-qa run config.toml --l2-only` | CLI executes without error; cross-table checks run against the converted Parquet |
| 3 | Verify the output report directory contains L2 results | Report generated |
| 4 | Verify no temp `.parquet` files are left behind in the system temp directory from the SAS conversion | No orphaned temp files |
| 5 | Re-run with `--l2-only` and a deliberately unknown table key in config (e.g., `custom_table = "/path/to/custom.sas7bdat"`) | CLI runs successfully with a warning about unknown table key; schema falls back to inference |

## Phase 4: Edge Cases

| Step | Action | Expected |
|------|--------|----------|
| 1 | Point a config table entry to an empty `.sas7bdat` file (0 rows) and run `--l2-only` | No crash; empty Parquet is created and cross-table checks handle it gracefully |
| 2 | Point a config table entry to a `.sas7bdat` with columns not in the SCDM spec and run `--l2-only` | Extra columns are preserved in the converted Parquet; no errors about unknown columns |
| 3 | Set `chunk_size = 1` in config options and run with a small SAS file | Conversion succeeds (one row group per row); verify no performance catastrophe for small files |
